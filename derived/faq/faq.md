# FAQ

___

# 为什么挂载触发器组件且编辑完节点图逻辑后，试玩不触发对应逻辑？

请确认：

①是否在【碰撞触发器组件】配置了触发区，且目标物进入了触发区

②触发区为初始生效

③正确设置作用对象

④已挂载节点图且有正确、完整触发逻辑，并且已经保存。

⑤请确认挂载的是【碰撞触发器】而非【碰撞触发源】

___

# 人物进入/离开碰撞触发器区域时，进入/离开者实体是玩家还是角色？为什么我获取不到进入/离开碰撞触发器的实体GUID？

玩家操纵人物进出触发区域时，事件节点中进入/离开者实体传出的是角色实体；

因此，进出触发区域的实体为角色时，进入/离开者实体GUID引脚传出值为空。

___

# 为什么实体高速穿过碰撞触发器时不触发效果？

实体高速通过较窄的碰撞触发器时，有概率无法触发碰撞触发器，建议奇匠根据需要调整碰撞触发器大小；

___

# 为什么我在【选项卡】中已经配置了触发区，但是在节点图中【进入/离开碰撞触发区】取不到事件？

因为【进入/离开碰撞触发区】只监听【碰撞触发器】的进出事件，对【选项卡】组件中的【触发选项区域】不监听。如需要额外监听玩家进入触发选项区域事件，需要额外配置一个【碰撞触发器】。

___

# 为什么我已经在计时器管理中配置了计时器，但是在节点图中动态开关/恢复、取值不生效？

A：计时器除了需要在计时器管理界面配置好后，还需要在对应的动态元件上添加【计时器】组件，并且手动选取需要挂载的对应计时器，才会被该元件的节点图调取并执行。

例：元件在倒计时5秒后打印字符串：“触发”

①计时器管理界面配置倒计时为5秒的全局计时器：

②元件挂载【全局计时器】，且勾选上刚刚创建配置好的倒计时：

③挂载带有全局计时器事件逻辑的节点图

④节点图内逻辑完整

⑤成功触发

___

# 计时器为倒计时/正计时时，节点图中【全局计时器触发时】事件节点什么时候会被触发？

当全局计时器类型为倒计时时，【全局计时器触发时】事件会在倒计时结束时触发（如倒计时5秒，会在进入游戏5秒后触发）。

当全局计时器类型为正计时时，【全局计时器触发时】不会被触发。

___

# 全局计时器什么时候开始计时？

全局计时器默认其挂载的元件被创建后立即启动。

___

# 为什么我在运动器组件中配置的向量数据（xyz），在实际试玩中的移动方向和预期不符？

在运动器组件中（以匀速直线为例），其速度向量中xyz为世界坐标。并且不管在实体摆放页面，元件本身如何旋转，在运动时都会基于已配置好的世界坐标来移动。

因此，建议不要通过元件本身的操纵杆（handle）来判断运动器中的xyz，而是主要参考运动预览中蓝色指示线来判断。

___

# 为什么我在元件上已经配置好了运动组件和节点图，但是配置未生效？

在“局内编辑器”中，如果“基础运动器组件”设置的生效时间过短，由于通信延迟存在，信息还未传达到节点图，运动就已经结束，会导致节点图无法生效；请奇匠将生效时间设置为大于0.1s的值；

___

# 力场器中逻辑力场球体和触发区域是什么关系？为什么角色已经进入逻辑力场球体范围，但是不会被力场影响？

角色必须进入扰动装置中的触发区域才会开始受到逻辑力场的影响。因此，如果想要玩家接触逻辑力场球体就会被牵引，其触发区域必须要比逻辑立场球体范围大。

例：如下图，角色没有踩上挑战起点元件中心时，是不会被力场牵引的，尽管力场范围已经将元件以及周围都涵盖了。

同理，如果进入触发区，但是没有进入逻辑力场球体范围，也不会触发牵引逻辑。

___

# 为什么我在【角色扰动装置】中已经配置了触发区，但是在节点图中【进入/离开碰撞触发区】取不到事件？

因为【进入/离开碰撞触发区】只监听【碰撞触发器】的进出事件，对【角色扰动装置】组件中的【触发区域】不监听，如需要额外监听玩家进入触发选项区域事件，需要额外配置一个【碰撞触发器】。

___

# 为什么在组件中选择跟随目标，下拉列表中没有我想要跟随的实体？

请确认想要跟随的目标实体已经被放置在【实体摆放】界面中。

___

# 为什么在节点图中选择跟随目标角色，但实际游玩不生效？

跟随运动器目标需要GUID来做定位，但角色本身没有GUID，导致功能不运行

___

# 自定义变量、节点图变量有什么不同？

自定义变量和节点图变量虽然都是变量，但是层级不同：

【自定义变量】挂载在实体上后，除了可以被自身节点图逻辑读取和修改外，还可以被其他实体挂载的节点图逻辑读取和修改；

【节点图变量】只能在单个节点图中存储并生效，不过可以通过公开变量的方式让其他节点图取到。

___

# 为什么我的自定义变量出入参不生效？读取不到/值都是0？

确定已经挂载自定义变量组件，并且其目标自定义变量的名称、数据类型都准确无误。保证从发出、查询、监听所有节点的数据类型都和目标自定义变量数据类型相等。

___

# 为什么变量已经变化了，但【自定义变量变化时】不生效？

首先【自定义变量变化时】只能监听它所挂载的元件身上的变量改变，请确保没有跨元件监听变量变化的设计；

其次，请重点确认【变化后值】【变化前值】的数据类型与想要监听的数据类型完全匹配！（点击小齿轮可以设置数据类型）如果想要监听的变量数据类型为字符串，但是【自定义变量变化时】中的【变化后/变化前值】为泛型/其他数据类型，会导致监听失败！

___

# 为什么修改自定义变量的名字后会和部分功能的关联会断开？

当修改自定义变量、能力单元等组件修改名字后，该组件在其相关界面（例如“交互界面编辑”界面）的关联关系需要重新配置。

___

# 为什么在特效资产中，有些之前见过的特效找不到？为什么有时候打开特效资产，发现内容不一样？

特效实际上分为限时特效和循环特效，两类特效的资产库是完全不同的。因此在设置特效的时候请确保特效资产类型为您需要的类型。

___

# 移动端试玩过程中，音效会异常消失？

移动端的性能优化逻辑导致，同时播放的音效数量有上限。

___

# 为什么我的命中检测不生效？

“命中检测”组件中，针对阵营和实体类型的筛选只会对“受击盒”命中层生效。

___

# 为什么我在元件库编辑元件后，切换到实体摆放界面，没有在实体列表里面找到我编辑过的元件？

在元件库编辑元件后，一定要点击保存，才会在实体摆放界面的自定义实体/造物（怪物）里找到。

# 我怎么获取不到角色的GUID？

角色本身没有GUID，因此请注意在节点图GUID参数获取中，不要获取角色的GUID，这会导致功能不运行。

___

# 玩家和角色之间是什么关系？

在局内编辑器中，【玩家】是服务器概念上的独立个体，即每个进入游戏的玩家，每名玩家的变量存储也都独立存在；【角色】是由玩家进行操作控制的特殊实体，在关卡中与各类组件进行交互的实体都是【角色】；不过需要注意，【玩家】有GUID，但是【角色】没有GUID。

___

# 我怎么让角色手持武器？

现阶段武器是通过特效来配置的，在角色的【特效】组件上挂载【循环特效】，在资产中选择“模型-单手剑/大剑”后，在挂接点上根据自身需求选择挂接点。

___

# 为什么我的角色复苏之后又立马被击败了？

关卡内角色的复苏逻辑是：先在原地复苏，再传送到复苏点；因此，在特定情况下，角色复苏后可能会被立即击败。

___

# 为什么我的技能命中目标数量变多之后，后面的目标无法被正常命中了？

技能同时命中的目标超过20个时，基于技能选中目标的逻辑，超过20个以外的目标将无法受到任何伤害。

___

# 在技能节点图的预览中，为什么只能预览到部分技能节点图中的执行效果？

在“千星沙箱”中，编辑技能相关节点时，仅支持预览部分“执行节点”的执行效果。
